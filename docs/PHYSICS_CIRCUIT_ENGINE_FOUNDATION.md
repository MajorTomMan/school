# School 物理模型、直流电路与电力内核

## 物理模型

物理课程不能只保存一条公式。`PhysicalModelSpecification` 同时声明：

- 物理量及量纲
- 方向是否重要
- 模型假设
- 可用方程
- 方程适用条件

模拟运行前，`ModelConditionState` 检查必要假设。例如理想抛体模型可以要求“忽略空气阻力”和“重力场均匀”。教材未声明的条件不能被系统偷偷补入。

## 电路图

`CircuitGraph` 把电路表示为节点和元件边，而不是图片像素。首批元件：

- 电阻
- 灯泡
- 独立电压源
- 独立电流源
- 导线
- 开关
- 电流表
- 电压表
- 电容
- 电感
- 二极管

电容、电感和二极管已有结构定义，但本轮不参与线性直流稳态求解，界面会明确说明而不是给出伪结果。

## 拓扑诊断

`CircuitTopologyValidator` 检查：

- 元件两端连接同一节点
- 悬空节点
- 电源外部开路
- 电源被理想导线短路
- 电流表与其他元件并联
- 电压表疑似串联
- 当前求解器不支持的动态或非线性元件

含严重错误的电路不会进入数值求解。

## 直流求解

`LinearDcCircuitSolver` 使用改进节点分析：

1. 为非接地点建立节点电压未知量。
2. 为独立电压源增加电流未知量。
3. 按元件向矩阵写入电导、源和约束。
4. 使用带主元选择的高斯消元求解。
5. 回算每个元件的电压、电流和功率。

支持：

- 一般线性电阻网络
- 多个独立电压源和电流源
- 串联与并联
- 开关闭合和断开
- 电表有限内阻模型
- 等效电阻
- 并联支路识别
- 灯泡功率与额定功率比

电源功率采用被动符号约定：负功率表示向外供能。

## 电学关系

`ElectricalRelations` 基于共享物理量和单位层实现：

```text
I = U/R
P = UI = I²R = U²/R
W = Pt
Q = I²Rt
```

输入必须具有正确量纲，结果使用安培、瓦特和焦耳表示。

## 边界

本轮不是 SPICE：

- 不做瞬态电容/电感积分
- 不做交流相量和阻抗
- 不做半导体器件非线性迭代
- 不做温度相关电阻曲线
- 不做真实灯丝热惯性

这些能力已有扩展位置，会在相应教材章节或更高阶段加入。当前目标是可靠覆盖初高中直流电路、电表、电功率和电能内容。
