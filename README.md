# School

一个只服务个人学习过程的 Android 原生 App。它不是网课平台，也不是简单的 PDF 阅读器；目标是把教材变成可以真正走完的学习路径：

```text
教材定位 → 动态讲解 → 自适应练习 → 错因诊断 → 到期复习
```

## 当前版本：0.18.1

当前版本已经建立从 PDF 直导、预制数学知识、本地 OCR、动态课程到数学题库的基础闭环：

- 黑、白、红、蓝、黄五色极简场景式 UI
- 小学、初中、高中、大学四级教育阶段
- 内置教材 PDF 浏览器，一次授权目录后可直接搜索全部子目录
- 首次教材导入不再强制完成教程，可以直接进入教材浏览器
- 多学科、多年级、学期教材独立管理
- 自动校验 PDF 文件头、页数与可读性
- 文件名、封面、版权页和目录 OCR 联合识别未知教材身份
- 自动推断教材印刷页与 PDF 索引偏移
- 十一册人教版数学教材预制知识包，覆盖二千零五十一页和二百八十个知识单元
- 匹配教材优先按 SHA-256 指纹加载预制目录、页码和原创动态课程，不依赖手机 OCR 生成知识点
- 未匹配教材继续使用本地 OCR、本地知识编译器以及可选的文本或视觉模型
- 中文 OCR 使用 Google Play 服务按需模型，不再把完整识别模型打包进 APK
- 数轴、相反数、绝对值、大小比较等主题使用专门动态场景
- 数学、方程、函数、几何、向量、数列、导数和概率统计等主题生成结构化步骤、误区与练习
- WorkManager 持久后台复制、校验、扫描、课程分析和原题线索提取
- OCR 日志保存原始文字、清洗结果、过滤原因、识别耗时和可疑文字
- 数学题库提供教材同步、薄弱强化、错题重练和综合练习
- 精确分数、表达式等价和一元一次方程确定性判题
- Room 保存作答、错误类型、知识点掌握度、错题和复习计划
- Android CI 自动测试、构建 APK，并更新 `dev-latest` 预发布版本
- 主分支开发 APK 使用稳定签名缓存，后续版本可以覆盖安装并保留应用数据

## 数学预制知识包

当前预制范围：

```text
初中数学：七年级上册、七年级下册、八年级上册、八年级下册、九年级上册、九年级下册
高中数学 A 版：必修第一册、必修第二册、选择性必修第一册、第二册、第三册
```

预制知识包不包含教材 PDF、教材图片或大段教材正文，只包含：

```text
PDF SHA-256 指纹
教材身份与页数
印刷页偏移
知识点标题与页码范围
School 原创的讲解、动态步骤、误区与练习模板
```

导入匹配 PDF 后，应用会优先使用这套稳定数据。PDF 仍保存在用户设备上，用于阅读教材原页和核对出处。未匹配教材不会被强行套用预制数据，会继续走普通扫描流程。

## 导入教材 PDF

在 App 中进入：

```text
学习阶段 → 学科 → 年级或学年 → 册次或学期 → 教材文件
```

首次可以授权 `Documents/教材` 等目录。之后 App 会递归扫描目录中的 PDF，按文件名推测阶段、学科、年级和册次，并将与当前教材槽位高度匹配的文件排在前面。导入教程不再阻塞第一次使用，可以直接选择教材文件。

导入时会依次执行：

```text
复制 PDF
→ 校验文件头、页数和 SHA-256
→ 查找预制知识包
→ 匹配成功：加载预制目录与动态课程，不启用 OCR
→ 未匹配：按需下载中文 OCR 模型，识别封面目录并执行本地知识编译
→ 建立课程、页面索引和题目线索
```

所选文件必须是真实 PDF。仅修改扩展名不会通过 `%PDF-` 文件头和 `PdfRenderer` 可读性校验。导入成功后，课程与题库可以返回教材对应印刷页核对原文。

## 可选 OCR 模型

0.18.1 起，中文 OCR 从随 APK 打包的 ML Kit 模型改为 Google Play 服务按需模型：

- 安装 App 和使用预制数学课程时不会下载 OCR 模型。
- 只有未知教材无法匹配预制知识包、确实需要扫描时才会触发下载。
- 首次 OCR 需要设备联网，并可能需要等待 Google Play 服务完成模型安装。
- 没有 Google Play 服务的设备仍可使用预制课程、题库、错题和复习，但不能对未知教材执行本地中文 OCR。

这个选择减少了 APK 自身占用，也让主要使用预制课程的安装保持轻量。

## APK 更新与签名

GitHub Actions 会缓存固定的开发调试证书。安装一次由 `master` 生成的 0.18.1 APK 后，后续主分支 APK 在证书缓存仍有效时可以直接覆盖安装：

```bash
adb install -r school-debug.apk
```

从此前随机签名的 0.17.0 或更早版本迁移到 0.18.x 时，可能仍需要最后卸载一次。之后不要混装 Pull Request 临时产物和主分支 `dev-latest` APK，应持续使用主分支发布的 APK 更新。

长期正式发布仍应把专用签名密钥保存在 GitHub Actions Secrets 或其他安全密钥存储中，不能把私钥提交到公开仓库。

## llama.cpp 配置

在 App 的「设置」中填写：

```text
接口地址：http://电脑局域网地址:7777/v1
模型名称：与 /v1/models 返回的 id 一致
API Key：局域网服务未启用鉴权时留空
```

手机和电脑需要处于可互访的网络中。数学预制课程与基础题库不依赖模型；模型不可用时仍可学习、练习和记录错题。

## 本地运行

使用支持 AGP 9.2 的 Android Studio 打开仓库并运行 `app`。也可以在已安装 Gradle 9.4.1 与 Android SDK 36 的环境中执行：

```bash
gradle :app:testDebugUnitTest :app:assembleDebug
```

仓库暂未提交 Gradle Wrapper 二进制文件，CI 会显式安装 Gradle 9.4.1。

## CI/CD 与自动 APK

每次 Pull Request 都会自动运行单元测试并编译 Debug APK。每次代码进入 `master` 且测试、编译成功后，工作流还会：

1. 恢复或生成稳定开发签名。
2. 生成 `school-debug.apk` 和 SHA-256 校验文件。
3. 上传为 GitHub Actions 构建产物。
4. 更新名为 `dev-latest` 的滚动预发布版本。
