## 修改点

- 教材导入入口只允许选择 PDF，压缩包、图片和其他文档不再出现在文件选择器中。
- 教材后台处理改为直接复制 PDF，不再要求用户预先制作包含清单与目录的教材压缩包。
- 新增 PDF 文件头校验，所选文件必须包含标准 `%PDF-` 文件头并可由系统 PDF 渲染器正常读取。
- 新增小学、初中、高中、大学四级教育阶段入口，教材、课程、题库和学习记录按阶段分别展示。
- 小学支持一年级至六年级，初中支持七年级至九年级，高中支持高一至高三，大学支持大一至大四。
- 不同教育阶段使用独立学科模板；大学阶段增加计算机、经济学和法学等课程领域入口。
- 高中和大学教材槽位使用上学期、下学期标签，小学和初中继续使用上册、下册。
- 新增 PDF 身份识别流程，综合文件名、封面、版权页和前二十四页 OCR 文字判断教材标题、学科、年级与册次。
- 识别到明确的学科、年级或册次冲突时停止导入，避免教材进入错误的年级槽位。
- 新增印刷页码偏移推断，通过页面底部 OCR 页码估算 PDF 索引与教材印刷页之间的偏移。
- 新增目录扫描算法，从目录页行文本中提取课程标题与起始页，并自动计算课程页码范围。
- 无法可靠识别目录时，按教育阶段使用不同页面跨度生成可继续分析的课程区段。
- 自动在应用私有目录生成内部清单、目录索引、教材身份记录和页面索引，旧版阅读、OCR、课程和题库流程继续复用。
- 封面和目录扫描得到的 OCR 页面会直接写入缓存，后续课程分析无需重复识别相同页面。
- PDF 复制、校验、教材身份识别、目录扫描、课程生成和保存继续使用持久后台任务，并在完成或失败后发送通知。
- 首次导入教程更新为 PDF 直导说明，明确本地 OCR、后台处理和文件限制。
- 版本升级到 0.15.0。

## 修复点

- 修复打开教材 PDF 所在目录后所有 PDF 都显示为不可选择的问题。
- 修复“导入教科书”实际只接受预制 ZIP 教材包、与界面含义不一致的问题。
- 修复普通教材 PDF 即使被传入后台也会因为缺少 `manifest.json` 而失败的问题。
- 修复教材只能按一至九年级展示、无法区分小学、初中、高中和大学的问题。
- 修复高中和大学教材被强行套用初中上下册层级的问题。
- 修复直接导入 PDF 时没有教材标题、目录和课程骨架的问题。
- 修复教材印刷页与 PDF 页索引存在封面偏移时课程跳转容易错页的问题。
- 修复明显选择错误学科或年级的 PDF 仍可能覆盖当前教材槽位的问题。
- 修复同一封面和目录页面在导入阶段与课程分析阶段重复 OCR 的问题。
- 修复旧版已安装教材缺少教育阶段字段时无法继续读取的问题；旧数据会根据年级自动归入对应阶段。
