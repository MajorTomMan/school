## 修改点

- 将项目 Java 工具链升级到 21，并通过 Gradle Foojay 工具链解析器在本地与 CI 环境自动准备一致的 JDK。
- 引入当前最新稳定版 Apache Lucene Kuromoji 10.5.0，作为日语形态素分析底座。
- 新增 School 自有 `JapaneseMorphologicalAnalyzer` 接口和 `JapaneseMorpheme` 数据模型，隔离课程、规则分析和 UI 对 Lucene 类型的直接依赖。
- 新增 Lucene Kuromoji 适配器，可返回表层词、原形、读音、发音、词性、活用类型、活用形与字符位置。
- 新增日语分词、助词词性、动词原形、礼貌体助动词、活用信息和位置范围测试。

## 修复点

- 修复现有日语分析只能依赖手写助词列表切分、无法可靠获得词性、原形、读音和活用信息的底层能力缺口。
- 避免业务层直接绑定特定第三方分词库，后续更换词典或分析器时无需重写课程和界面。
- Android 产物暂时继续保持 Java 17 字节码基线，避免仅为构建依赖而无必要地提高设备运行要求。
