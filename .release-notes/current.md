## 修改点

- APK 改为纯云端课程运行时，不再打包教材目录、章节标题、课程正文、例题、小结、习题或教材 PDF。
- 结构化课程数据和完整课程 ZIP 由稳定的 GitHub `course-latest` Release 发布，APK 默认读取其中的 `manifest.json`。
- 教材 PDF 暂时保存在 Google Drive，作为课程清单中的外部文件按需下载，不进入 GitHub 仓库、课程 ZIP 或 Release。
- 完整课程 ZIP 解压后会补齐外部教材；旧缓存中摘要一致的 PDF 会直接复用，只有缺失或变化时才重新下载。
- 更新计划会把外部 PDF 的传输体积计入全量成本，继续支持文件级增量更新和完整包回退。
- 教材 PDF 下载后会校验文件大小、SHA-256、PDF 文件头和 202 页页数，再与课程数据一起原子切换到 `active`。
- 教材中心只展示已经从云端下载并验证的教材，不再提供本地选择、绑定或替换 PDF。
- 首次打开且尚未安装课程时，先提示课程包与教材 PDF 的下载体积，只有用户确认后才开始下载。
- 选择“稍后”时本次启动不联网下载课程；下次仍未安装课程时会再次提示。
- 已安装课程的设备不会重复显示首次下载提示，冷启动后继续自动检查增量更新。

## 修复点

- 完整课程 ZIP 不再重复携带十余 MB 的教材 PDF，课程内容更新时不会重复发布或下载未变化的教材。
- 新增 `inFullPackage` 文件声明，明确区分 Release 包内资源和 Google Drive 外部资源。
- 完整安装时若 ZIP 缺少外部文件，会从旧缓存安全复用或通过清单地址补齐，不再直接判定课程包损坏。
- GitHub Actions 自动构建、验证并覆盖发布 `manifest.json`、课程 JSON 和不含 PDF 的完整 ZIP。
- 未安装课程时不再在后台静默开始首次下载，避免在用户未确认的情况下消耗网络流量。
- APK 版本提升至 0.21.0；课程清单最低版本设为 22，旧客户端不会错误安装新的分离式课程包。
