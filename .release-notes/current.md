## 修改点

- 新增语文、英语、日语、物理、化学三十四本预制教材目录，与现有十一册数学共同覆盖四十五本教材。
- 新增一千零二十七个非数学可学习节点；加上现有二百八十个数学知识单元，共一千三百零七个预制课程节点。
- 预制资产仅保存 PDF 指纹、教材身份、目录层级、页码范围、学习角色和 School 自有教学描述，不把约 1.09 GB 教材 PDF 或教材正文打进 APK。
- 使用教材自带 PDF 书签恢复单元、章、节、课、实验、写作、听说、项目和整本书阅读等真实层级；日语教材从“学习内容一览”恢复课程。
- 新增语文、语言和理化学科专属预制分析模板，分别强调文本证据、语境交际、条件规律和实验过程。
- 数学继续使用原有专属确定性知识包与题库，不被通用模板替换。
- 新增高中必修、选择性必修及上中下册稳定卷册 ID，解决原有上册/下册二元槽位无法容纳高中多册教材的问题。
- 高中多卷册选择改为两列布局，避免大量册次挤在同一行。
- 课程同步器读取预制目录路径，构建真实 `MODULE / UNIT / CHAPTER / LESSON / TOPIC` 父子树，不再把所有课程平铺在教材下。
- 导入 PDF 后继续通过 SHA-256 或规范化文件名匹配预制包，匹配成功后替换为预制目录和分析结果。
- 新增全学科预制索引完整性测试，验证教材数、课程数、槽位与指纹唯一性、页码边界、路径节点类型和卷册 ID。
- 新增“一次函数”交互样板课，按教材顺序从对应值、列表、描点和连线进入函数图像，暂不提前引入斜率和截距。
- 一次函数新增按要求量生成输入框的关系计算器：已知 `x` 求 `y` 时只输入 `x`，已知 `y` 求 `x` 时只输入 `y`；关系包含更多变量时自动显示其余所有已知量输入框。
- 新增公式与方程验证器，可识别式子中的多个变量，动态生成输入框，显示代入过程、等号两边的值并判断是否成立；首版支持数字、字母、括号与加减乘除。
- 新增“牛顿第一定律”交互样板课，按教材顺序比较阻力明显、阻力较小和理想无阻力三种情形，并显示具有明暗、滚动和停止效果的小球动画。
- 两节交互课程新增带页码的教材原文短引用、教材内容依据、思想过程、背景知识、常见误区及教材原页核对入口。
- 新增课程知识权限、统一诊断结果和动态变量关系三组基础模型，从代码层限制课程只能启用当前章节允许的概念、操作与组件。
- 新增“交互能力实验室”主导航入口，集中展示复平面、空间点正交投影、化学方程式配平和植物细胞结构标注四个独立样板。
- 复平面样板支持实部、虚部输入和点位同步；三维坐标样板支持 `x/y/z` 输入、视角调整和 `xy` 平面投影。
- 化学样板通过反应前后氢、氧原子计数验证 `H₂ + O₂ → H₂O` 的配平；生物样板提供教学示意图、结构热点与名称说明。
- 交互能力实验室改用统一扁平分区、下划线输入与结果区，移除重复的圆角卡片堆叠。
- 化学配平改为直接绘制 H₂、O₂ 和 H₂O 分子及原子连接关系，不再使用统计横条表达原子数。
- 化学样板只输入左侧 H₂、O₂ 系数，根据原子守恒自动生成右侧 H₂O 系数；左侧比例不合法时显示原因，不再要求手动填写生成物。
- 植物细胞补充细胞壁，并按细胞壁、细胞膜、细胞质及内部结构重绘分层和选中引线。
- 新增英语与日语共用语言学习引擎，支持文本规范化、多种可接受表达、词语顺序、英语词形、日语助词、日语活用和语体诊断。
- 交互能力实验室新增英语样板，验证第三人称单数词形、句子结构和 `I am / I'm` 等价表达。
- 交互能力实验室新增日语样板，支持汉字与假名读音切换，并验证移动助词和礼貌体动词活用。
- 开发版 APK 改为使用仓库固定的公开开发签名，并在 CI 中同时输出文件 SHA-256 与签名证书 SHA-256。
- 新增科学计算共用内核，使用精确有理数保存分数与有限小数，避免根号、π 和单位换算过早转换为浮点数。
- 新增根式、π、e、虚数单位、幂、分数、隐式乘法和变量表达式 AST，支持基础精确化简与近似求值。
- 新增七个 SI 基本量纲、常见导出量纲、常用数学物理化学单位、精确单位换算、有效数字和测量不确定度模型。
- 交互能力实验室新增“科学内核”样板，可直接验证 `√72`、`2π+3π`、`72 km/h` 和 `1 kWh` 等表达式与单位换算。
- 新增精确多项式四则运算、求值、导数和线性因式除法。
- 新增一次方程、二次方程、判别式、一次不等式、定义域约束和函数采样模型，求根过程保留精确分数与根式。
- 新增二维与三维向量、点、直线、圆、平面、投影、相交和几何变换模型。
- 新增结构化数学证明步骤和依赖关系校验，能够发现前向引用和重复事实。
- 科学内核实验室新增数学深化页，可输入二次方程系数并查看精确根、判别式步骤和向量投影。
- 新增物理模型规格、变量、方程和必要假设，模拟运行前可以验证模型条件是否完整。
- 新增电路图结构与电源、电阻、灯泡、开关、导线、电流表、电压表、电容、电感和二极管元件模型。
- 新增电源短路、开路、悬空节点、电流表并联和电压表疑似串联等拓扑诊断。
- 新增基于改进节点分析的线性直流电路求解器，计算节点电压、支路电流、元件功率、等效电阻和灯泡亮度比例。
- 新增欧姆定律、电功率、电能和焦耳热的精确物理量关系。
- 科学内核实验室新增串联与并联电路样板，同步显示节点电压、电流、功率和亮度。
- 新增包含一百一十八个元素身份、课堂常用相对原子质量和常见氧化态的周期表教学数据。
- 新增通用化学式解析器，支持元素、下标、多层括号、结晶水、离子电荷和物质状态，并计算原子组成、相对式量和质量分数。
- 新增通用化学方程式模型，分别保存反应物、生成物、系数、状态、可逆性和反应条件。
- 新增基于有理数矩阵零空间的方程式配平，结果自动化为最小正整数比并复核原子数与总电荷守恒。
- 新增离子方程式旁观离子约去，以及摩尔质量、浓度、稀释、限量试剂、理论产率和产率百分比计算。
- 交互能力实验室新增“化学内核”样板，可解析复杂化学式并配平多物种反应。
- 新增有机分子图，保存原子、形式电荷、显式与隐式氢、单键、双键、三键、芳香键和连接关系。
- 新增受限结构式解析，支持分支、环编号、芳香原子和简单方括号原子，并生成分子式与价态诊断。
- 新增碳骨架、最长碳链、双三键位置、官能团检测、分子图同构和构造异构判断。
- 新增羟基、醚键、醛基、酮羰基、羧基、酯基、氨基、酰胺基、卤素、碳碳多键和芳香环识别。
- 新增带元素与电荷守恒校验的教材限定有机反应模板，首个样板验证乙烯催化加成。
- 新增确定性小分子二维布局和有机化学实验室，可绘制键级、比较乙醇与二甲醚并高亮官能团。
- 启动初始化改为首帧优先：预制教材安装和掌握度趋势监听在 Compose 首屏建立后转入 IO 后台执行。
- 新增预制教材初始化版本标记与 `SchoolStartup` 分阶段耗时日志；版本一致时不再解压、解析全部课程目录。
- 新增应用内在线升级，展示修改点与修复点，下载并验证 GitHub `dev-latest` 的完整 APK 后调用系统安装器。
- CI 为每次构建生成递增版本号、签名更新清单和清单数字签名，主分支发布 APK 后立即更新滚动开发 Release。
- 新增可选 FCM 数据推送：`dev-latest` 上传完成后主动通知已安装 App，App 再下载并验签正式更新清单。
- 前台收到推送时直接显示升级弹窗，后台收到推送时执行一次 WorkManager 检查并显示新版本通知；启动检查和每日检查继续作为兜底。
- 设置页新增应用版本、更新通道、即时提醒配置状态、自动检查、仅 Wi-Fi 下载和手动检查入口。
- 新增全学科教材预制流水线、交互课程架构、语言学习引擎、开发签名、科学内核、启动性能和即时更新说明文档。
- 版本升级到 0.20.0。

## 修复点

- 修复高中英语七册、语文五册及理化多册教材无法用“年级 + 上下册”唯一表示的问题。
- 修复预制目录只有扁平课程列表，教材真实单元、章节和活动结构无法进入课程树的问题。
- 修复所有非数学课程只能使用同一种泛化讲解模板的问题。
- 修复新增预制教材必须先绑定 PDF 才能进入课程的问题；现在未绑定原书也可以学习、记录掌握度和复习。
- 修复教材目录中实验、写作、听说、语法、项目等节点被误当成普通知识课的问题。
- 修复一次函数图像在参数较大时绘制到坐标区域外的问题，现在直线被严格裁剪在绘图区内。
- 修复一次函数概念页提前使用斜率、截距和斜率三角形的问题。
- 修复牛顿第一定律页面提前使用摩擦系数、牛顿第二定律公式和速度—时间图的问题。
- 修复一次函数验证实验同时使用滑块修改 `x`、`y`，与实际求值题只提供已知量的交互方式不一致的问题。
- 修复交互能力实验室中化学原子计数条和植物细胞边框使用错误圆角参数导致 Android 编译失败的问题。
- 修复复数输入值可超过坐标范围、显示数值与图上位置不一致的问题，实部和虚部现在限制为 `-5～5`。
- 修复化学系数过大导致分子超出绘图区的问题，左侧两个系数现在限制为 `1～4`。
- 修复植物细胞仅用红点表示结构、细胞壁和细胞核层级难以理解的问题。
- 修复 GitHub 主分支、不同 PR 和对话下载 APK 因 Actions 缓存作用域不同而使用不同 debug 证书、无法互相覆盖安装的问题。
- 修复三个启动 ContentProvider 在主线程同步读取材料库、解压预制索引、解析一千三百零七个课程节点并创建分析数据库，导致冷启动首屏迟迟不能出现的问题。
- 修复应用内更新功能此前只存在于功能分支、主分支 Release 未上传签名清单而无法形成真实升级闭环的问题。
- 修复仅依赖六小时前台检查和二十四小时后台检查，导致 GitHub 新 APK 发布后不能及时被设备发现的问题。
