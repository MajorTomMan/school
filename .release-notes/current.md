## 修改点

- 新增独立学科定义层，内置语文、数学、英语、日语、科学、物理、化学、生物、历史、地理、思想政治、计算机、编程、经济学、法学、音乐、美术、体育与健康十八个常见学科。
- 学科通过可组合能力描述阅读、写作、词汇、语法、听力、发音、数值判定、表达式判定、分步判定、图形、实验、时间线、地图、论证和编程等能力。
- 新增课程体系 `Curriculum`，不再让某一本教材承担学科和课程身份。
- 新增任意深度课程目录树 `CurriculumNode`，支持根、阶段、等级、学期、课程、模块、单元、章节、课和主题节点。
- 课程树使用稳定父子 ID，支持排序、祖先、后代、扁平展示、旧课程映射和循环校验。
- 新增独立知识点 `KnowledgePoint`，课程节点通过多对多引用绑定主要知识、辅助知识和复习知识。
- 新增知识关系图，支持前置、组成、相关、等价和扩展关系；数学基础知识预置真实前置依赖。
- 新增统一学习资源与资源绑定，PDF、预制课程、题库、文档、图片、音频、视频和 Web 资源可以绑定课程节点或知识点及页码范围。
- 教材库变化时自动将现有 `TextbookSlot`、教材、二百八十个数学课程和页码转换为课程树、知识点与资源绑定。
- 旧 `GeneratedLesson.id` 保存为课程节点的 `legacyLessonId`，已有课程状态、复习记录和页面跳转继续兼容。
- 新增通用知识掌握度表，主键改为学科与知识点，同一知识在不同教材和课程路径之间共享掌握度。
- 数学题库继续按教材保存尝试、错题和原页来源，但读写全局知识掌握度。
- Room 数据库升级到版本 3，增加学科、等级体系、课程、节点、知识点、知识关系、资源、绑定、全局掌握度和节点进度十二组实体。
- 数据库升级时将旧数学掌握度按知识点聚合迁移，保留旧尝试、错题和教材来源。
- 课程完成、开始和需要复习状态继续写入旧 DataStore，同时双写课程节点进度。
- “路径”页面改为展示真实课程父子结构、知识点、资源绑定状态和当前课程的前置知识。
- 首次选择学科教材时不再强制进入多页导入教程，直接进入教材浏览器。
- 中文 OCR 继续使用 Google Play 服务按需模型，预制课程不会触发 OCR 下载。
- 新增课程树顺序、深度、旧课程映射、多前置知识、循环校验、引用完整性、全学科 ID 唯一性和稳定知识 ID 单元测试。
- 新增完整课程与知识架构文档。
- 版本升级到 0.19.0。

## 修复点

- 修复课程体系长期绑定“学科 + 年级 + 上下册”，无法自然表达大学课程、语言等级、职业课程和自学路径的问题。
- 修复目录只有教材与课程两层，无法保存单元、章节、模块和主题父子关系的问题。
- 修复一个知识点只能隐含属于一节课，无法被多个课程路径复用的问题。
- 修复知识依赖被教材顺序代替，无法表达多个前置知识和跨章节关系的问题。
- 修复掌握度以教材和知识点作为联合主键，切换教材后同一知识重新从零开始的问题。
- 修复教材 PDF 被当作课程存在条件，没有原书时预制课程可能从教材索引中过滤的问题。
- 修复预制数学课程在连续写入教材索引时可能只保留最后一本的问题。
- 修复课程路径页标题硬编码为“有理数”，无法反映当前学科和课程体系的问题。
- 修复首次教材导入教程阻塞正常教材选择流程的问题。
- 修复清除学习记录时未清除新课程节点进度和全局知识掌握度的问题。
