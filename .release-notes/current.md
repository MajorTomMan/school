## 修改点

- 对用户提供的十一册数学教材进行离线扫描，共覆盖初中六册、高中五册、二千零五十一页 PDF。
- 根据 PDF 书签目录、教材页码和章节层级整理出二百八十个核心知识单元，不再依赖手机 OCR 猜测章节标题。
- 新增预制数学知识包，应用内只保存教材指纹、知识点标题、页码范围和原创教学结构，不内置教材 PDF 或教材正文。
- 导入数学 PDF 后优先计算 SHA-256 指纹；与预制教材精确匹配时，直接使用预制目录、页码偏移和动态课程。
- 文件指纹无法匹配时可使用规范化文件名尝试匹配，但仍会校验教育阶段、年级、册次和 PDF 页数。
- 已经导入的匹配数学教材会在应用启动或教材库刷新时自动升级为预制知识包，无需重新选择 PDF。
- 预制知识包覆盖正负数、数轴、相反数、绝对值、代数式、方程、不等式、函数、几何、向量、数列、导数、概率统计等数学主题。
- 数轴、相反数、绝对值和大小比较继续使用专门动画场景；其他主题按知识类别生成结构化过程、核心直觉、误区、步骤和练习。
- 课程顶部会明确显示“预制教材知识包”，便于与本地 OCR、文本模型和视觉模型结果区分。
- 重新分析预制数学教材时会恢复经过校验的预制课程，不再用低质量 OCR 结果覆盖。
- 新增预制目录完整性测试，检查十一册教材指纹唯一、页码范围有效以及二百八十个知识单元全部可读取。
- GitHub Actions 新增稳定开发签名缓存；同一稳定签名生成的后续主分支 APK 可以覆盖安装并保留教材、学习记录和设置。
- 构建日志会输出开发签名的 SHA-256 证书指纹，便于确认两个 APK 是否能够覆盖更新。
- 版本升级到 0.18.0。

## 修复点

- 修复数学教材课程质量过度依赖手机 OCR，扫描版、公式密集页或复杂排版容易生成无意义原文的问题。
- 修复目录 OCR 错误时章节被拆成固定页数区段，课程标题无法对应真实知识点的问题。
- 修复高中数学必修第二册缺少可用文字层时难以生成稳定课程结构的问题。
- 修复同一本数学教材在不同手机上因 OCR 差异产生不同知识点和课程内容的问题。
- 修复已经导入的数学教材无法自动使用后续新增预制知识数据的问题。
- 修复重新分析教材可能用 OCR 结果覆盖预制课程的问题。
- 修复 GitHub Actions 每次在全新构建机器生成不同调试证书，导致新 APK 经常无法直接覆盖旧版的问题。
- 修复更新 APK 时必须先卸载应用，进而丢失应用私有目录中教材与 OCR 缓存的长期风险。
